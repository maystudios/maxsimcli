<objective>
Implement phase archive sweep: when a phase completes, move its directory to `.planning/archive/<milestone>/`, collapse its ROADMAP.md section to a status line, prune its decisions/blockers from STATE.md. Add `get-archived-phase` CLI command for agent access to archived data.
</objective>

<execution_context>
This is the core infrastructure for context rot prevention. The existing `phaseCompleteCore` in phase.ts marks checkboxes and updates fields but does NOT archive. We are extending the phase completion lifecycle with archive capability.

Key locked decisions from CONTEXT.md:
- Archive triggers on phase complete, not via separate command
- Full sweep: dir + ROADMAP + STATE in one atomic operation, one commit
- User confirmation required: preview first, then execute on confirm
- Preview/execute is two separate tool calls (preview returns summary, workflow agent confirms, second call executes)
- Only prune decisions matching `- [Phase N]:` pattern; leave untagged decisions alone
- Empty sections after pruning get "None." placeholder
</execution_context>

Add `archivePhaseExecute(cwd: string, phaseNum: string, outcomeSummary: string): Promise<CmdResult>`:
- Re-read all files fresh (do NOT cache from preview â€” see pitfall #6)

- Prune STATE.md blockers: same pattern for Blockers section
- Collapse ROADMAP.md: (a) remove the `### Phase N:` detail section entirely, (b) replace the `- [ ]` checklist line with `- [x] Phase N: Name -- outcomeSummary` (pitfall #3 â€” handle both formats)
