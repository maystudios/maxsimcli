<action>
**1. Implement stale context detection** in `state.ts`:

Add `cmdDetectStaleContext(cwd: string): Promise<CmdResult>`:
- Read ROADMAP.md and identify completed phases (lines matching `- [x]` checkbox pattern)
- Read STATE.md and scan Decisions and Blockers sections for references to completed phases (lines matching `- [Phase N]:` where N is a completed phase number)
- Also scan for any `Phase N` mentions in free text of decisions/blockers sections that reference completed phases
- Return `cmdOk` with:
  ```typescript
  {
    stale_references: Array<{ section: string; line: string; phase: string }>;
    completed_phases: string[];
    clean: boolean; // true if no stale references found
  }
  ```
- If no stale references found, return `clean: true` with a message

**2. Update milestone complete** in `milestone.ts`:

Modify `cmdMilestoneComplete` to:
- Convert sync `fs` calls to async `fsp` for consistency (use `import { promises as fsp } from 'node:fs'`)
- Use `archivePath` from core.ts (from Plan 01) instead of legacy `.planning/milestones/` path
- Before reset, snapshot STATE.md and ROADMAP.md to `.planning/archive/<milestone>/STATE.md` and `ROADMAP.md`
- After snapshot, reset STATE.md to a clean template:
  ```markdown
  # Project State

  ## Project Reference

  See: .planning/PROJECT.md (updated YYYY-MM-DD)

  ## Current Position

  Milestone: <new milestone name>
  Phase: 0 of ? (not started)
  Status: planning
  Last activity: YYYY-MM-DD

  ## Performance Metrics

  No plans executed yet in this milestone.

  ## Accumulated Context

  ### Decisions

  None.

  ### Pending Todos

  None.

  ### Blockers/Concerns

  None.

  ## Session Continuity

  Last session: YYYY-MM-DD
  ```
- Commit all changes (archive snapshots + reset STATE.md) in one git commit

**3. Add CLI dispatch** in `cli.ts`:

Add `detect-stale-context` dispatch entry -> calls `cmdDetectStaleContext(cwd)`

**4. Export** `cmdDetectStaleContext` from `packages/cli/src/core/index.ts`.
