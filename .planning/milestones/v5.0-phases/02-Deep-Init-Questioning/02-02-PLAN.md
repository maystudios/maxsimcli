Follow the structure from 02-RESEARCH.md Pattern 5, with these sections:

1. **Header:** Title "Conventions", with `{{date}}` and `{{source}}` template placeholders (source = "new-project init" or "init-existing scan + user confirmation")

2. **Tech Stack** section: Table with columns Category, Technology, Version, Purpose. Include example rows as commented-out placeholders showing the expected format.

   - Pattern field (exceptions / Result types / error codes / error boundaries)
   - Standard flow description block
   - "What to catch vs let propagate" guidance
   - Include guidance comment: "Agents use this to write consistent error handling across all tasks."

5. **Testing** section: Table with columns Aspect, Standard. Include rows for: Framework, Test location, Naming convention, Coverage target, What to test per task.

   - Greenfield instructions: "Populate from research agent recommendations + questioning confirmations"
   - Brownfield instructions: "Populate from codebase/CONVENTIONS.md scan results + user confirmation of each convention"
   - Note: "Generated at init. Update when conventions change. See Phase 3 (Agent Coherence) for lifecycle updates."

7. **Footer:** `*Conventions {{generated_or_confirmed}}: {{date}}*`

The template should be 50+ lines. Use `{{placeholder}}` syntax consistent with other templates in the templates/ directory.
