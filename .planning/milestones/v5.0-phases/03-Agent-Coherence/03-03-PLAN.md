**Part B: Add 5 agent-level init functions to init.ts.**

Add these AFTER the existing workflow-level init functions (after the last `cmdInit*` function, before the module's end). Each function follows the established pattern:

1. Accept `cwd: string` and `phase: string | undefined` (phase optional for debugger)
2. Load config via `loadConfig(cwd)`
3. Find phase via `findPhaseInternal(cwd, phase)` if phase provided
4. Assemble a role-filtered context object matching the interface
5. Collect codebase docs: scan `.planning/codebase/` directory for .md files, return array of relative paths
6. Return `cmdOk(result)`

**Key role-specific filtering:**
- `cmdInitExecutor`: Include executor_model, verifier_model, branching_strategy, parallelization. Exclude researcher_model, planner_model.
- `cmdInitPlanner`: Include planner_model, checker_model, research_enabled, plan_checker_enabled. Exclude executor_model, branching_strategy, parallelization.
- `cmdInitResearcher`: Include researcher_model, brave_search. Exclude executor_model, verifier_model, branching_strategy, parallelization, plan_checker_enabled.
- `cmdInitVerifier`: Include verifier_model. Exclude executor_model, researcher_model, branching_strategy, parallelization.
- `cmdInitDebugger`: Include debugger_model (resolve for 'maxsim-debugger'). Exclude executor_model, verifier_model, branching_strategy, parallelization.

**Codebase docs helper:** Create a small helper function `listCodebaseDocs(cwd: string): string[]` that scans `.planning/codebase/` for .md files and returns relative paths. Used by all 5 init commands. If directory does not exist, return empty array.

Export all 5 new functions from init.ts.
