Read the full executor agent file. Add a new `<wave_review_protocol>` section. This section should be placed AFTER the task execution steps and BEFORE the SUMMARY.md creation step.

**The section must implement:**

1. **Unconditional two-stage review after all wave tasks complete:**
   No model profile check. No conditional. Always runs.

2. **Stage 1 - Spec-Compliance Review:**
   - Collect inline context for spec-reviewer:
     - Task specs (action, done criteria, files) for ALL tasks in this wave (copy from PLAN.md)
     - Modified files list: `git diff --name-only HEAD~{commit_count}` (where commit_count = number of task commits in this wave)
     - Plan frontmatter requirements list
   - Spawn `maxsim-spec-reviewer` as a subagent with all inline context
   - Parse review output: extract frontmatter using pattern matching for `status:`, `critical_count:`, `warning_count:` fields
   - If status is FAIL:
     - Fix the issues identified in the review
     - Re-stage and commit fixes
     - Re-run spec review (retry 1)
     - If still FAIL: fix again, commit, retry (retry 2)
     - If still FAIL after retry 2: output REVIEW BLOCKED structured response with stage name, attempt count, failing issues, and 3 options (fix manually, skip review, abort). STOP and wait for user.

3. **Stage 2 - Code-Quality Review:**
   - Collect inline context for code-reviewer:
     - Modified files list (same as stage 1, updated after any fix commits)
     - CONVENTIONS.md content if it exists (read from `.planning/CONVENTIONS.md` or `.planning/codebase/CONVENTIONS.md`)
     - Test results: run `npm test 2>&1 | tail -20` if package.json exists
   - Spawn `maxsim-code-reviewer` as a subagent with all inline context
   - Same retry logic as stage 1 (max 2 retries, then block)

4. **Review results recording:**
   After both stages complete (PASS or skipped), record in a review section that will be included in SUMMARY.md:
   ```
   ## Review Cycle
   - Spec: {PASS/FAIL/SKIPPED} ({retry_count} retries)
   - Code: {PASS/FAIL/SKIPPED} ({retry_count} retries)
   - Issues: {critical_count} critical, {warning_count} warnings
   ```

5. **Continuation mode handling:**
   Add a note in the section: "When resuming from checkpoint (continuation mode), review covers ALL tasks in the plan. Re-read the full PLAN.md and pass all tasks to reviewers, not just the post-checkpoint tasks. This is because checkpoint decisions may affect earlier work."

6. **Gap-closure plan handling:**
   Add a note: "This review protocol applies to ALL plans including gap-closure plans. No exceptions."

**IMPORTANT:** If there is any existing review-related conditional (checking model_profile or quality), REMOVE the conditional and replace with the unconditional protocol. The research warns about Pitfall 4: "Do not try to patch the condition -- rewrite the section to be unconditional."

Also update the executor's inline context checklist section (if one exists) to explicitly list what must be passed to each reviewer. If no such section exists, add it within the wave_review_protocol.
</action>
