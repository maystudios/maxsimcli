# Plan 05-01: Discuss Command and Workflow (FLOW-01)

<objective>
Create the `/maxsim:discuss` command and workflow -- a unified entry point that triages user-described problems, ideas, and bugs into the right size (quick todo or phase). This replaces the need for users to choose between add-todo, add-phase, or discuss-phase for new items.

The command handles both new items (user describes a problem) and existing todos (user references a pending todo). It uses adaptive questioning (2-3 questions, expand if complex), always confirms routing with the user before filing, and offers next actions after filing.
</objective>

**Key distinction from discuss-phase:** The `/maxsim:discuss` command triages an UNKNOWN problem into the right size. `/maxsim:discuss-phase` gathers implementation decisions for a KNOWN phase. These are complementary, not overlapping.

**Anti-patterns (from RESEARCH.md):**
- Do NOT create a new agent -- discussion is orchestrated by the workflow using AskUserQuestion directly
- Do NOT duplicate todo creation logic -- call existing maxsim-tools.cjs commands (init todos, generate-slug, etc.)
- Do NOT duplicate phase creation logic -- route to existing `phase add` command
- Do NOT modify existing command files (add-todo.md, discuss-phase.md) -- GUARD-02 applies
- Do NOT auto-route without user confirmation -- every routing decision MUST go through AskUserQuestion

<action>
Create the discuss workflow following the structure of existing workflows (discuss-phase.md, add-todo.md). This is a markdown workflow file -- no TypeScript code.

**Workflow structure (use XML step tags like other workflows):**

**Step 1: init_context**
- Load project state: `node ~/.claude/maxsim/bin/maxsim-tools.cjs state-load --raw`
- Check planning_exists. Hard stop if false.
- Load pending todos: `node ~/.claude/maxsim/bin/maxsim-tools.cjs init todos`
- Parse user input (argument from command invocation, if any)

**Step 2: detect_existing_todo**
- If user provided a reference (argument), search pending todos for title/slug match
- If match found, use AskUserQuestion to confirm: "Found existing todo: [title]. Want to discuss this one?" with options Yes / No, this is new
- If no match or no argument, proceed as new item

**Step 3: gather_context (adaptive questioning)**
- Use AskUserQuestion to understand the problem/idea/bug
- Start with a question about what the user wants to accomplish
- Follow up with 1-2 more questions based on answers (adaptive depth)
- If answers reveal complexity, ask more. If simple, move to triage quickly.
- Apply thinking-partner behaviors: challenge vague answers, surface assumptions, propose alternatives
- Reference `@./references/thinking-partner.md` for the thinking partner behavior set

**Step 4: triage**
- Based on gathered context, propose a size classification via AskUserQuestion:
  - "Quick fix (todo)" -- Small, well-defined, file as todo
  - "Needs a phase" -- Too big for a todo, add to roadmap
  - "Let me explain more" -- User wants to provide more context (loop back to step 3)
- CRITICAL: Always present this as a question with options, never auto-route
- Include a brief explanation of WHY the system suggests its classification

**Step 5a: file_as_todo (if user chose todo)**
- Generate slug: `node ~/.claude/maxsim/bin/maxsim-tools.cjs generate-slug "[title]"`
- Create todo file in `.planning/todos/pending/` following the exact pattern from add-todo.md workflow
- Use the same frontmatter format: area, priority, created date
- Infer area from discussion context (same logic as add-todo.md)
- Check for duplicates against existing todos before creating
- Write the file using the Write tool
- Commit: `node ~/.claude/maxsim/bin/maxsim-tools.cjs commit "docs: add todo [slug]" --files .planning/todos/pending/[slug].md`

**Step 5b: file_as_phase (if user chose phase)**
- Use AskUserQuestion to confirm phase details: name, brief goal description
- Show preview of what would be added to ROADMAP.md via AskUserQuestion: "This would add Phase [N]: [name] to the roadmap. Proceed?"
- Only after user confirms: `node ~/.claude/maxsim/bin/maxsim-tools.cjs phase add "[name]"`
- Commit: `node ~/.claude/maxsim/bin/maxsim-tools.cjs commit "docs: add phase [name]" --files .planning/ROADMAP.md .planning/phases/[dir]`

**Step 6: offer_next_action**
- After filing (either path), use AskUserQuestion to offer next actions:
  - If filed as todo: "Want to work on it now? (/maxsim:quick)" / "Save for later" / "Check all todos (/maxsim:check-todos)"
  - If filed as phase: "Want to discuss it? (/maxsim:discuss-phase [phase])" / "Want to plan it? (/maxsim:plan-phase [phase])" / "Save for later"
- User always chooses -- no auto-start

**Tool mandate (copy from discuss-phase.md):**
Include the AskUserQuestion tool mandate block at the top of the workflow: every single question MUST use AskUserQuestion, never plain text responses. This is critical for the workflow to function correctly.

**Success criteria section:**
Add a success_criteria section at the end listing checkboxes for: item filed, routing confirmed by user, next action offered, git commit created.
</action>
