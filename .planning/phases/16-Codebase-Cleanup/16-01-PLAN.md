<objective>
Eliminate the entire `packages/cli/src/adapters/` directory by inlining its useful functions into install modules, remove the vestigial `RuntimeName` and `AdapterConfig` types from core, clean up install flow references to multi-runtime abstractions, and update CLAUDE.md to reflect Claude Code-only reality. The build and all tests must pass after every task.
</objective>

<execution_context>
MAXSIM was originally multi-runtime (Claude, OpenCode, Gemini, Codex). Since v2.0 it is Claude Code only. The adapter abstraction layer still exists but only has one implementation. This plan removes that dead abstraction.

The critical ordering is: create new home for functions FIRST, redirect imports SECOND, delete adapters THIRD, clean types FOURTH, update docs FIFTH. This prevents build breakage at every step.

Read the research file for full details on every function, import, and consumer:
@.planning/phases/16-Codebase-Cleanup/16-RESEARCH.md
</execution_context>
